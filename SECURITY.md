# Security Policy

## Supported Versions

We currently provide security updates only for the latest release on the `main` branch.

| Version | Supported          |
| ------- | ------------------ |
| latest (main) | :white_check_mark: Yes |
| older versions/tags | :x: No             |

## Reporting a Vulnerability

**This is primarily an educational/research tool for market analysis and stochastic simulation. It is not intended for production trading or managing real funds.**

If you believe you have found a **security vulnerability** (e.g. code execution issues, unsafe deserialization, credential leakage in examples, dependency vulnerabilities, etc.), please report it responsibly.

**Preferred way**:
- Open a **private** vulnerability report → https://github.com/ASAPUI/MARGINS/security/advisories/new  
  (GitHub will create a private security advisory visible only to maintainers)

**Alternative** (if you prefer):
- Email: [alirayanessabri@gmail.com]  
  Subject: "Potential security vulnerability in MARGINS"

Please include:
- Description of the issue
- Steps to reproduce
- Affected versions / commit hash
- Any proof-of-concept code (if safe to share)
- Potential impact

We will acknowledge receipt within 3–7 days and aim to respond with assessment / fix timeline.

**Important**: Do **not** create public issues or discuss potential vulnerabilities publicly until we have addressed them.

Thank you for helping keep MARGINS safe.
